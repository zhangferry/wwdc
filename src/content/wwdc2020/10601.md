## 一句话判断

如果你是创意工作者而非程序员，这场 Session 展示了如何用 Reality Composer 和 AR Quick Look 零代码创建 AR 体验——Apple 正在努力降低 AR 的创作门槛。

## 这场 Session 讲了什么

这场 Session 面向设计师、艺术家和创意人员，展示了 Apple 提供的一整套 AR 创作工具链。演讲者从一个核心观点出发：AR 内容创作不应该只有程序员才能做。Apple 通过 Reality Composer、Reality Converter 和 AR Quick Look 构建了一条从创意到交付的低代码甚至零代码路径。

首先介绍了 Reality Composer——一个 macOS 和 iOS 上的可视化 AR 场景编辑器。你可以在其中导入 3D 模型、添加材质和动画、设置交互行为（如点击触发动画），然后直接在设备上预览效果。2020 年的更新增加了对 USDZ 格式的更多支持，以及新的动画和行为模板。

然后讲解了 Reality Converter，这个工具让艺术家可以将常见的 3D 文件格式（OBJ、PLY、ABC 等）转换为 Apple 生态所需的 USDZ 格式，同时可以预览和调整材质效果。演讲者通过一个家具展示的实际案例，演示了从设计师的 3D 建模文件到可在线浏览的 AR 体验的完整工作流。

最后讨论了如何将创建的 AR 内容嵌入到网页和 App 中，利用 AR Quick Let 实现即点即看的 AR 体验。

## 值得深挖的点

- **USDZ 作为统一格式的意义**：Apple 推动 USDZ（基于 Pixar 的 USD 格式）作为 AR 内容的标准格式。这意味着不管你用什么工具创作，最终都要落地到 USDZ。理解这个格式的特性（如基于物理的材质、骨骼动画）对于优化 AR 内容质量至关重要。
- **AR Quick Look 的商业价值**：对于电商场景，AR Quick Look 提供了最低摩擦的 AR 购物体验——用户在 Safari 中点击一个链接就能看到 3D 产品在自己空间中的样子，无需安装任何 App。这对转化率的提升有直接帮助。

## 代码片段

```html
<!-- 在网页中嵌入 AR Quick Look -->
<a rel="ar"
   href="https://example.com/models/chair.usdz">
    <img src="chair-thumbnail.jpg"
         alt="点击查看 AR 模型">
</a>
<!-- iOS Safari 会自动识别 rel="ar" 链接
     并打开 AR Quick Look 预览 -->
```

```swift
// 在 App 中使用 AR Quick Look
import QuickLook

class ARViewController: UIViewController, QLPreviewControllerDataSource {
    func numberOfPreviewItems(in controller: QLPreviewController) -> Int {
        return 1
    }

    func previewController(
        _ controller: QLPreviewController,
        previewItemAt index: Int
    ) -> QLPreviewItem {
        // 指向 USDZ 模型文件的 URL
        let modelURL = Bundle.main.url(
            forResource: "furniture",
            withExtension: "usdz"
        )!
        return modelURL as QLPreviewItem
    }
}
```

```swift
// 使用 Reality Composer 生成的场景
import RealityKit

func loadRealityComposerScene() {
    // 加载 Reality Composer 导出的 .rcproject
    let anchor = try! Experience.loadScene()

    // 可以通过编程方式修改场景元素
    if let model = anchor.entities.first?.children.first {
        // 调整模型位置和缩放
        model.position = [0, 0, -1]
        model.scale = [0.5, 0.5, 0.5]
    }

    arView.scene.anchors.append(anchor)
}
```

## 最佳实践

- 3D 模型的面数控制在合理范围内，移动端 AR 场景建议单模型不超过 100K 面
- 使用基于物理的材质（PBR）提升真实感，但注意纹理分辨率不要过大以节省内存
- 为 AR Quick Look 体验提供缩略图和尺寸信息，帮助用户预期 AR 内容的大小
- 动画设计考虑空间因素——AR 中的动画速度感知与屏幕上的 2D 动画不同

## 还有什么值得关注

- Reality Composer 后来在 Xcode 中直接集成了编辑器，不再需要单独下载
- 如果你需要程序化生成 AR 内容，可以搭配 "Explore ARKit 4" 和 "What's new in RealityKit" 学习编码方式
- Apple 的 AR 艺术馆（AR Art Gallery）项目展示了艺术家如何用这些工具创作沉浸式体验
