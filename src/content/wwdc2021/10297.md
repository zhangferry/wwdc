## 一句话判断

如果你在帮企业或学校批量管 iOS 设备，Apple Configurator 2 的自动化工作流能让你的「刷机」效率从小时级降到分钟级。

## 这场 Session 讲了什么

Apple Configurator 是 macOS 上的设备管理工具，用于批量配置 iOS/iPadOS/tvOS 设备。这个 Session 展示了 macOS Monterey 上 Apple Configurator 2 的更新，以及它如何配合新的「设备备份和恢复」功能来加速部署流程。

核心更新是「备份和恢复」工作流的改进。以前批量部署设备需要逐台连接 USB、擦除、安装配置文件、安装 App——即使有自动化脚本也很耗时。macOS Monterey 支持把一台已经配置好的设备的完整状态（系统版本、App、配置文件、用户数据）备份为一个镜像，然后批量恢复到其他设备上。配合 automator 或命令行工具，你可以实现「连上 USB 就自动刷机」的全自动流程。

Session 还介绍了 Apple Configurator 对「声明式设备管理」（Declarative Device Management）的支持。这是一种新的设备管理协议，设备主动向 MDM 服务器声明自己的能力和状态，而不是被动接受命令。这减少了 MDM 的轮询开销，提高了设备管理的实时性。

## 值得深挖的点

**自动化工作流的命令行工具链。** Apple Configurator 提供了 `cfgutil` 命令行工具，可以脚本化几乎所有设备管理操作。你可以写一个 bash 脚本：检测连接的设备 -> 擦除 -> 恢复备份 -> 安装配置文件 -> 安装 App -> 断开连接。在批量部署场景（比如开学前给 500 台 iPad 刷机）中，这种自动化能把每台设备的配置时间从 15 分钟压缩到 3-5 分钟。`cfgutil` 支持 `prepare`、`restore`、`install-app`、`get-deviceType` 等子命令，配合 `--json` 输出可以和其他工具链集成。

**备份加密和安全考量。** 批量恢复备份时，备份文件可能包含敏感数据（Wi-Fi 密码、VPN 配置、企业证书）。Apple Configurator 支持对备份文件加密，但你需要注意：加密密码不会存储在 Configurator 中，忘记密码意味着无法恢复备份。在企业部署中，建议用 Keychain 或密码管理器存储备份密码，而不是写在脚本里。另外，恢复到新设备时，某些数据（如 Apple Pay 卡片、iCloud 登录状态）无法通过备份迁移，需要用户手动设置。

## 代码片段

```bash
# 使用 cfgutil 命令行工具批量管理设备

# 列出所有连接的设备
cfgutil list

# 准备设备（擦除并安装指定 iOS 版本）
cfgutil --ecid DEVICE_ECID prepare \
    --install-path "/path/to/IPSW.ipsw" \
    --supervise \
    --organization "公司名称"

# 从备份恢复设备
cfgutil --ecid DEVICE_ECID restore \
    --backup-path "/path/to/backup" \
    --backup-password "加密密码"

# 安装 App
cfgutil --ecid DEVICE_ECID install-app \
    --path "/path/to/App.ipa"
```

```bash
# 批量自动化脚本示例
#!/bin/bash
# 自动配置所有连接的设备

# 获取所有设备 ECID
DEVICES=$(cfgutil list --format json)

# 遍历每台设备
for ECID in $(echo $DEVICES | jq -r '.[].ecid'); do
    echo "正在配置设备: $ECID"

    # 擦除并准备
    cfgutil --ecid $ECID prepare \
        --supervise \
        --organization "学校" \
        --dep-profile

    # 恢复标准备份
    cfgutil --ecid $ECID restore \
        --backup-path "/shared/standard-backup"

    # 安装额外的 App
    cfgutil --ecid $ECID install-app \
        --path "/shared/apps/Numbers.ipa"

    echo "设备 $ECID 配置完成"
done
```

## 最佳实践

- 在批量部署前，先用一台设备做完整的测试恢复流程，确认备份包含所有必要的配置和 App
- 为不同用途的设备创建不同的备份模板（教室用、办公室用、展示用），避免在恢复后再手动调整
- 定期更新备份镜像以包含最新的系统版本和安全补丁——恢复旧版本后还需要做 OTA 升级很浪费时间
- 企业环境中的设备监督（Supervision）要在准备阶段就启用，后期追加需要擦除设备

## 还有什么值得关注

- Apple Configurator 现在支持通过 Wi-Fi（而不是只有 USB）管理已监督的设备
- 声明式设备管理（Declarative Device Management）在 macOS Monterey 和 iOS 15 上可用，是 MDM 协议的未来方向
- Finder 在 macOS Monterey 中也可以管理设备（恢复模式、系统更新），Apple Configurator 提供的是更高级的批量管理能力
