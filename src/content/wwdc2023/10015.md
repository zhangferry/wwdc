## 一句话判断

App Store 预订购终于支持按地区独立管理了——你可以先在一个地区发布 App，然后在其他地区开放预订购，这对灰度发布和分地区运营策略来说是关键能力。

## 这场 Session 讲了什么

Session 介绍了 App Store Connect 中预订购功能的三个重要更新：

1. **按地区管理预订购**：全新未发布的 App 可以选择特定地区开放预订购，不再是全有或全无的选择。
2. **已发布 App 的预订购**：已经在某些地区上架的 App，可以在其他新地区开放预订购。这支持"软发布"策略——先在一个市场试水，反响好再在更多地区通过预订购造势。
3. **预订购的编辑与管理**：可以修改已有预订购的发布日期、立即发布、或取消预订购。

时间规则上，全新未发布 App 的预订购日期最远可设到 180 天后，已发布 App 扩展到新地区时最远可设到 365 天后。

Availability 页面新增了详细的地区状态视图，能一目了然地看到每个国家/地区的发布状态（已上架、预订购中、不可用）。

此外，预订购 App 现在也支持自定义产品页面（Custom Product Pages）和产品页优化测试（Product Page Optimization），这些测试的数据可以在 App Analytics 中追踪。

## 值得深挖的点

**"先发布、再预购"的模式**是这次更新的核心价值。之前预订购只能用于完全未发布的 App，现在你可以在加拿大先上线，收集用户反馈和运营数据，确认产品没问题后，再在美国等大市场开放预订购。这种策略在游戏和社交类 App 中很常见——先用小市场验证核心指标，再用预订购在大市场蓄水。

**365 天的预购窗口期**对已发布 App 来说相当宽裕。你可以提前整整一年在大市场建立预购页面，配合营销活动持续积累预购用户。到期时批量推送通知+自动下载，相当于一次有准备的"发布日"事件。

**预订购与自定义产品页的组合**意味着你可以为不同地区、不同受众制作不同的预购页面，配合不同的营销素材做 A/B 测试。这在多地区发布时能显著提升转化率。

## 代码片段

预订购本身通过 App Store Connect 操作，不涉及代码。但相关的 API 和营销链接值得了解：

```swift
// 通过 App Store Connect API 管理预订购
// 查询 App 可用性（包括预订购状态）
// GET /v1/apps/{id}/appAvailability

// 创建预订购可用性
// POST /v1/appAvailabilities
// {
//   "attributes": {
//     "availableInNewTerritories": true
//   },
//   "relationships": {
//     "app": { "data": { "id": "你的AppID" } },
//     "territoryAvailabilities": {
//       "data": [
//         { "id": "USA", "attributes": { "preOrderEnabled": true, "releaseDate": "2024-03-01" } }
//       ]
//     }
//   }
// }
```

营销徽章和链接：

```swift
// 在你的网站或营销材料中使用本地化的预购徽章
// 链接格式：
// https://apps.apple.com/us/app/id你的AppID

// 预购用户在发布日会收到通知，App 自动下载到设备
// 无需用户手动操作
```

## 最佳实践

- 制定分地区发布计划时，先用小市场验证核心指标，再在大市场开放预订购。
- 预购页面尽早发布，配合社交媒体和广告投放持续积累预购用户。
- 利用自定义产品页为不同地区的预购页面展示不同的截图和描述，提高本地化转化率。
- 在 App Analytics 中持续跟踪各地区的预购数量和转化率，及时调整营销策略。
- 发布日期一旦确定就尽量避免推迟，预购用户对延期发布的容忍度很低。
- 预购发布后不要再改为取消——已预购的用户体验会受到负面影响，且同一地区无法再次开放预购。

## 还有什么值得关注

- 所有新功能都支持 App Store Connect API，可以实现预订购管理的自动化。
- 预购 App 在 App Store 中可被搜索到，还有机会出现在 Today、Games、Apps 标签页的推荐中。
- App Analytics 提供按日期和按地区维度的预购数据，方便评估各地区营销效果。
- 已发布地区和预订购地区可以同时存在，互不影响——已发布地区正常下载，预订购地区等待发布。
- 本地化的预购徽章素材可以在 Apple 的营销资源页面下载。
