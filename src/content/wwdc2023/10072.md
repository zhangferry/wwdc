## 一句话判断

这是空间计算平台的设计入门课——从窗口到沉浸，从人体工学到视觉真实性，这节课把最基本的设计原则讲透了。

## 这场 Session 讲了什么

Session 从五个维度讲解了空间计算平台的设计原则：熟悉感、以人为本、维度、沉浸感和真实性。

窗口采用全新的玻璃材质，提供对比度并与环境融合。窗口会随着用户移动自动转向面对他们，确保始终可读。窗口尺寸应该基于内容选择——Safari 用高度看更多网页，Keynote 用宽度适配演示。tab bar 和 toolbar 可以延伸到窗口之外。

以人为本的设计要求将内容放在视野中央，使用横向布局匹配宽视野。内容放置应考虑人体工学——默认沿自然视线放置，鼓励远距离交互。避免把内容放在身后或极高/极低的位置。不要将内容锚定到视野中（会感觉粘滞），而是锚定在空间中。

沉浸感是渐进式的。从窗口中的 2D 内容，到带有 3D 元素的 Volume，再到 Mixed 模式的部分沉浸，最后到 Full 模式的完全沉浸。每种程度的沉浸都有适合的场景。

## 值得深挖的点

Points 系统在空间计算平台上仍然适用。界面元素用 points 指定大小，窗口随距离自动缩放——远的变大，近的变小。这保证了内容在任何距离下都可读可用。

数字或人（Digital + Physical）的概念很重要。当有人走过你的沉浸空间时，系统会自动将他们从数字内容中"拉"出来，确保你始终能看到身边的人。这是平台设计的核心人文关怀。

## 代码片段

```
空间设计核心原则：

1. 熟悉感
   - 使用用户已知的 UI 元素（侧边栏、标签、搜索）
   - 玻璃材质窗口与空间融合
   - 窗口自动转向面对用户

2. 以人为本
   - 重要内容放在视野中央
   - 使用横向布局
   - 内容放在略远于手臂的距离
   - 不要锚定到视野，锚定到空间
   - 创建静坐式体验，不需要走动

3. 维度
   - 渐进式沉浸：窗口 → Volume → Mixed → Full
   - 保持默认窗口较小，避免遮挡视野
   - 慎用多窗口

4. 沉浸感
   - 根据内容选择沉浸程度
   - 让用户控制沉浸程度
   - 渐进式引导进入沉浸

5. 真实性
   - 数字内容看起来真实
   - 尊重物理世界的规则
   - 自动处理人物遮挡
```

## 最佳实践

- 窗口尽量小，避免遮挡太多视野
- 理想情况下保持 app 界面在单一窗口中
- 使用 points 设计界面，系统自动处理距离缩放
- 避免锚定内容到用户视野，让它在空间中有固定位置
- 让用户通过按住 Digital Crown 来重新居中内容

## 还有什么值得关注

- 参考 "Design Spatial User Interfaces" 了解界面设计细节
- 窗口尺寸应根据内容类型选择（高或宽）
- 空间音频是沉浸体验的重要组成部分
- 人物在数字内容前自动获得优先级
