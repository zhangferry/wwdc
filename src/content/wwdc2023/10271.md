## 一句话判断

这不是一篇代码导向的技术 talk，而是一堂关于空间音频设计哲学的大师课。

## 这场 Session 讲了什么

声音设计师 Danielle Price 从实践角度讲解了如何在空间计算平台上设计声音。核心概念是空间音频源（Spatial Audio Sources）——声音会根据你放置的位置发出，系统会根据你所在空间的尺寸和材质自动添加混响效果。

Session 涵盖了三个层面的声音设计：UI 交互音效、沉浸式环境音和录制/混音技巧。在 UI 音效方面，每个交互都应该有微妙的反馈音，比如虚拟键盘的每次按键都从对应位置发出声音。对于高频出现的音效，应该稍微随机化音高和振幅，避免机械感。

在环境音方面，Session 分享了 Mount Hood 环境音的设计过程。实地录制时发现，真实的声音环境并不总是理想的——排水系统的噪音可能盖过所有自然声音。作为设计者，你有权"策划"最好的现实，让声音与视觉效果相得益彰。

## 值得深挖的点

录制和混音的工作流程值得注意。环境底音使用全向沉浸式麦克风录制整个空间的空气感，而特定声音则使用高灵敏度指向性麦克风捕捉。录制覆盖全天候，寻找噪声污染最少的地点。

Session 强调了一个关键设计原则：好的 UI 声音应该足够微妙，只提供必要的反馈。虽然你可以在自己的 app 中加入一些个性，但要确保它与整体 UI 的感觉一致。比如 Photos app 的声音设计没有使用"窗户打开"或"投影仪幻灯片"这样过于具象的声音，而是选择了与系统音效风格一致且体现深度感的设计。

## 最佳实践

- UI 音效要微妙，只提供必要的反馈
- 高频触发的音效应随机化音高和振幅
- 环境音录制应覆盖不同时段，寻找最佳声音条件
- 策划最好的现实，不必完全忠实于录音
- 使用专业音效库作为补充资源
- 声音应与视觉风格保持一致

## 还有什么值得关注

- Find My 的 ping 音效是空间音频在日常生活中的典型应用
- 声音设计需要考虑多模态体验——文字和图像应与声音协同
- 环境音的明暗两个版本应分别设计
