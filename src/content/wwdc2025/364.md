## 一句话判断

这是 WWDC25 的导览视频——100 多场 Session 的导航入口。别跳过，因为今年每个 Session 多了一个能直接复制代码到 Xcode 的功能，很多人还不知道。

## 这场 Session 讲了什么

WWDC25 的欢迎视频，本质是一个"展会地图"。它快速勾勒了今年最重要的几个方向：Liquid Glass 全新设计、Apple Intelligence 的 on-device 大模型、Xcode 26 的 AI 编程辅助、Swift/SwiftUI 的性能和并发改进、Metal 4、游戏生态等。

视频时长很短（10 分钟），信息密度不高，核心价值在于告诉开发者"今年有什么、该去看哪些 Session"。几个值得注意的细节：

- 每个 Session 今年都有了**文字摘要**，可以直接跳到关键内容
- 可以**从 Session 中直接复制代码到 Xcode**，不用再手打
- 新增了**在线小组 Lab**，Apple 工程师主持，支持提问和投票
- 一对一 Lab 预约仍然可用

## 值得深挖的点

**Session 文字摘要的实用价值**

往年看 WWDC Session 是纯视频体验——要么从头看到尾，要么拖进度条找关键点。今年每个 Session 有了 written summary，可以快速判断这个 Session 是否值得花时间看完，或者只看自己关心的部分。对于中文开发者来说，配合翻译工具使用效率更高。

**代码直接复制到 Xcode**

这个功能听起来小，但体验差别很大。以前看 Session 需要暂停、截图、手敲代码。现在可以直接选中代码块复制，粘贴到 Xcode 里就能跑。Apple 终于把这个基础体验做对了。

**在线小组 Lab 是个新渠道**

往年一对一 Lab 是 WWDC 现场专属福利。今年新增的在线 group lab 降低了参与门槛——即使你没去现场，也可以向 Apple 工程师提问。支持 upvote 机制，热门问题会优先回答。

## 代码片段

这个 Session 没有代码。它的价值是指引你去找到有代码的 Session。

## 最佳实践

1. **用这个视频做你的 Session 清单起点**。今年 100+ 场 Session，不可能全看。先看这个视频了解全貌，再根据自己 app 的技术栈筛选必看场次
2. **善用文字摘要功能**。在 developer.apple.com 上看 Session 时，先扫摘要再决定要不要看完整视频
3. **申请在线 Lab**。如果你的 app 有具体的适配问题（比如 Liquid Glass 自定义控件的兼容性），直接问 Apple 工程师比自己摸索高效得多

## 还有什么值得关注

- 如果你想快速了解今年所有重大更新，直接看 Session 101 Keynote 和 102 Platforms State of the Union
- Session 365 "New things on the way from Apple" 是面向消费者的更新概览，和这场形成互补
- Apple Developer Forums 和 Meet with Apple Developer Activities 也是持续获取帮助的渠道
- 今年 WWDC 的 sample code 质量明显提升，很多 Session 配套了可直接运行的 demo 项目
