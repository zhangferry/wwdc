## 一句话判断

这场 Session 最值得关注的一件事是：Apple 终于给 Siri 做了独立 App 并支持对话历史，这标志着 Siri 从“系统级语音助手”正式向“独立对话 AI 产品”转型。不过别被骗了，这根本不是一个技术 Session，而是一个时长 0 秒、充满电影旁白和音效的纯营销 Hype Video（宣传视频）。

## 这场 Session 讲了什么

这视频就是苹果放在 WWDC26 首页的“画大饼”预告片。从 iOS 18 引入 Apple Intelligence 开始，Siri 一直被吐槽只是个套壳，这次苹果直接把它做成了独立 App（dedicated app），让你能像用 ChatGPT 一样回顾历史对话。同时，Image Playground 从之前的卡通插画风，升级到了能生成“照片级真实（photorealistic）”图像，可以直接做壁纸和 Contact Posters（联系海报）。

对开发者来说，这意味着你的 App Intents 需要为更复杂的“多轮对话”和“屏幕上下文”做准备。Siri 不再只是执行单个指令的工具，而是一个能帮你策划看球派对、计算食物卡路里的全能 Agent（智能体）。

## 值得深挖的点

### Siri 独立 App 与多轮对话上下文

苹果终于给 Siri 做了独立 App，这背后的技术债其实非常大。以前 Siri 是系统级 UI，依附于 Spotlight 或锁屏，生命周期极短。现在变成独立 App，意味着它需要维护长期的 Conversation History（对话历史）和全局上下文。

为什么现在才做？因为端侧大模型（On-device LLM）的上下文窗口（Context Window）和内存占用终于达到了可用水平。设计上的 trade-off 在于，独立 App 会让 Siri 变得更“重”，用户可能不会像以前那样随口唤醒，而是需要“打开 App 去聊天”。和竞品相比，苹果的优势在于系统级权限（比如直接读取屏幕内容、操作其他 App），但坑在于，如果你的 App 没有良好地暴露 App Intents 和 UI 状态，Siri 在独立 App 里就多轮询问时，根本无法获取你 App 内的深层数据。

### Image Playground 迈向 Photorealistic（照片级真实）

从生成卡通图片到生成照片级真实图像，算力需求是指数级增长的。iOS 18 的 Image Playground 为了保端侧运行，只能生成特定风格的插画。iOS 27 敢承诺 photorealistic，说明苹果在 A 系列/M 系列芯片的 NPU（神经网络引擎）上做了巨大的底层优化，或者引入了更高效的 Private Cloud Compute（私有云计算）调度。

好的一面是，开发者可以通过 API 直接生成高质量素材，不用再依赖第三方生图接口。可能的坑是，照片级真实的图像极易引发深度伪造（Deepfake）和版权问题，苹果肯定会在 API 层面加入极其严格的 C2PA（内容出处和真实性联盟）水印和 NSFW（不适宜工作场所）过滤，这会导致你的合法请求被误杀的概率大幅上升。

## 代码片段

由于这是个 0 秒的预告片，没有官方代码。以下基于 iOS 18 的 API 合理推演 iOS 27 的概念代码。

### 1. 查询 Siri 对话历史 (概念 API)

场景：在你的 App 中获取用户最近与 Siri 的对话上下文，以便提供连贯的服务。

```swift
// 假设 iOS 27 提供了 SiriContext API
import AppIntents

struct FetchRecentSiriContextIntent: AppIntent {
    static var title: LocalizedStringResource = "获取最近的 Siri 上下文"
    
    func perform() async throws -> some IntentResult & ReturnsValue<String> {
        // 请求读取 Siri 历史对话的权限
        let context = try await SiriContextManager.shared.requestRecentContext(limit: 5)
        
        // 提取用户最近的意图和实体
        let lastQuery = context.interactions.last?.utterance ?? "无"
        return .result(value: lastQuery)
    }
}
```

坑：这种跨 App 的上下文读取必定需要用户显式授权，且系统可能会在后台对敏感数据进行脱敏（Redaction），拿到的数据可能是不完整的。

### 2. 生成照片级真实图像 (概念 API)

场景：使用 Image Playground 生成真实的联系人海报背景。

```swift
import ImagePlayground

func generatePhotorealisticPoster(for user: User) async throws -> UIImage {
    let configuration = ImagePlaygroundConfiguration(
        style: .photorealistic, // iOS 27 新增的照片级真实风格
        aspectRatio: .square
    )
    
    // 添加概念提示词和 C2PA 签名要求
    let concept = ImagePlaygroundConcept.text("专业商务人像，自然光")
    
    // 调用系统级生图 API
    let result = try await ImagePlaygroundSession.generate(
        concepts: [concept],
        configuration: configuration
    )
    
    return result.image
}
```

坑：`.photorealistic` 风格极大概率会强制走 Private Cloud Compute，在弱网环境下会直接超时，必须做好降级到本地 `.illustration` 风格的 fallback（回退）逻辑。

## 最佳实践

- **已有项目的迁移策略**：立刻审查你现有的 App Intents。Siri 变成独立 App 后，用户会在一个沉浸式界面里和你的 App 交互。如果你的 Intent 返回的只是简单的文本或基础 Snippet（代码片段），体验会非常割裂。赶紧用 `AppEntity` 和自定义的 `IntentView` 把 UI 丰富起来。
- **新项目的采用建议**：把 Siri 当作一个真正的 Agent（智能体）来设计。不要只暴露“打开某个页面”的 Intent，要暴露“查询状态”、“执行多步操作”的细粒度 Intent，让 Siri 能够自己组合调用。
- **实战中容易踩的坑**：不要假设 Siri 的上下文是永久的。系统会因为内存压力随时清理对话历史，你的 App 必须自己维护关键状态，不能依赖 Siri 帮你记住上一步操作。

## 还有什么值得关注

- **Safari Notify Me**：Safari 终于原生支持网页状态变更通知，不用自己写 Push Notification（推送通知）后端了。
- **密码一键修复**：系统级的 compromised passwords（泄露密码）自动更新，开发者需要确保自己的账号设置页面支持 AutoFill（自动填充）的密码更新流程。
- **Siri 屏幕感知**：视频里提到计算食物卡路里，暗示 Siri 的屏幕感知（Screen Awareness）能力大幅增强，能直接看懂第三方 App 的 UI 元素。
